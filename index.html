// 03 Reference

All commands

Complete command reference with syntax and examples.

Command & Syntax Description Role
!suggest <message>
e.g. !suggest Add a music bot
Submit a suggestion to the server's configured voting channel. The bot posts an embed with 👍 👎 reactions and a sequential suggestion number.
USER
!feedback <type> <message>
e.g. !feedback bug Buttons not working
Submit typed feedback. Types: general, bug, idea, compliment. Logged to the feedback channel with a colored embed.
USER
!ticket
e.g. !ticket
Sends the persistent support ticket creation panel with a button. Users click the button to open a private ticket channel.
USER
!help
e.g. !help
Displays the full help menu listing all available commands for the server.
USER
Command & Syntax Description Role
!show_config
e.g. !show_config
Displays all current server configuration — prefix, channels, categories, and roles.
ADMIN
!set_prefix <prefix>
e.g. !set_prefix /
Changes the command prefix for the server. All commands will then use the new prefix.
ADMIN
!set_suggestion_channel <#channel>
e.g. !set_suggestion_channel #suggestions
Sets the channel where community suggestions are posted with vote reactions.
ADMIN
!set_feedback_channel <#channel>
e.g. !set_feedback_channel #staff-feedback
Sets the channel where all submitted feedback is logged for staff review.
ADMIN
!set_resolved_channel <#channel>
e.g. !set_resolved_channel #closed-tickets
Sets the channel where resolved ticket transcripts are logged when tickets are closed.
ADMIN
!set_staff_role <@role>
e.g. !set_staff_role @Staff
Sets the role that gets permissions to view all support ticket channels.
ADMIN
!set_ticket_admin_role <@role>
e.g. !set_ticket_admin_role @Moderator
Sets the role notified via ping when a new support ticket is opened.
ADMIN
!set_ticket_category <Name or ID>
e.g. !set_ticket_category Support
Sets the default fallback category for ticket channels when no specific category applies.
ADMIN
!set_new_ticket_category <Name or ID>
e.g. !set_new_ticket_category New Tickets
Sets the category where freshly opened tickets appear before staff picks them up.
ADMIN
!set_working_ticket_category <Name or ID>
e.g. !set_working_ticket_category In Progress
Sets the category tickets move to when staff begins actively working on them.
ADMIN
// 04 Getting Started

Live in minutes

Four steps from zero to a fully configured bot.

01

Add the Bot

Click "Add to Discord" and authorize Sx2 Suggestions on your server.

02

Set Channels

Run !set_suggestion_channel and !set_feedback_channel to route events.

03

Configure Roles

Use !set_staff_role to grant ticket access and !set_ticket_admin_role for pings.

04

Go Live!

Run !ticket to post the panel. Members can now open tickets and submit suggestions instantly.
